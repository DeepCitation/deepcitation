# Plan: Remove XML `<cite />` Citations, Default to Numeric

## Context

DeepCitation supports two citation formats: numeric (`[N]` + JSON block) and XML (`<cite />`). The numeric format is the only one generated by prompts, is more token-efficient, and doesn't require normalization. The XML format is legacy dead weight — LLMs struggle with attribute escaping, and the normalization code is 500+ lines of workarounds.

This plan removes all XML cite code from the `deepcitation` package and renames the numeric functions to be the default (dropping the "Deferred" prefix). External consumers in `packages/web/` will break and be fixed in a follow-up PR.

## Step 3: Refactor Renderers to Numeric Format

All 5 renderers currently use `CITE_TAG_REGEX` + `parseCiteAttributes()` + `buildCitationFromAttrs()` to parse raw `<cite />` strings. Refactor each to accept `ParsedCitationResult` and render from pre-parsed citations.

### New input contract

```typescript
// Before: renderCitationsAsHtml(rawStringWithCiteTags: string, options)
// After:  renderCitationsAsHtml(parsed: ParsedCitationResult, options)
```

### New rendering pattern (replaces regex-replace)

```typescript
const segments = parsed.visibleText.split(parsed.splitPattern);
// segments = ["text", "[1]", "more text", "[2]", "end"]

let citationIndex = 0;
const output = segments.map(segment => {
  const match = segment.match(/^\[(\d+)\]$/);
  if (!match) return escapeHtml(segment); // plain text

  citationIndex++;
  const citationKey = parsed.markerMap[Number(match[1])];
  const citation = citationKey ? parsed.citations[citationKey] : undefined;
  if (!citation) return segment; // unknown marker, keep as-is

  const verification = verifications[citationKey] ?? null;
  const status = getCitationStatus(verification);
  return renderVariant(citationIndex, citation, status, options);
}).join("");
```

### Per-renderer changes

**`src/rendering/html/htmlRenderer.ts`**
- Remove `CITE_TAG_REGEX`, `parseCiteAttributes`/`buildCitationFromAttrs` imports
- Change `renderCitationsAsHtml(input: string, ...)` → `renderCitationsAsHtml(parsed: ParsedCitationResult, ...)`
- Replace `safeReplace(input, CITE_TAG_REGEX, ...)` with split-and-rebuild loop
- Variant functions (`renderHtmlCitation`) stay unchanged — they accept structured params

**`src/rendering/github/githubRenderer.ts`**
- Same structural change as HTML renderer
- `renderGitHubCitation()` variant functions unchanged

**`src/rendering/slack/slackRenderer.ts`**
- Same structural change
- `renderSlackCitation()` variant functions unchanged

**`src/rendering/terminal/terminalRenderer.ts`**
- Currently uses `safeMatch()` + manual replace loop (for ANSI color handling)
- Change to split loop, apply ANSI coloring per segment
- `renderTerminalCitation()` equivalent logic unchanged

**`src/markdown/renderMarkdown.ts`**
- Has LOCAL copies of `parseCiteAttributes()` and `buildCitationFromAttrs()` — delete them
- Remove local `CITE_TAG_REGEX`
- Change `renderCitationsAsMarkdown(input: string, ...)` → `renderCitationsAsMarkdown(parsed: ParsedCitationResult, ...)`
- Same split-and-rebuild pattern

### Convenience overload

Each renderer should also accept a raw `string` for ergonomics, auto-parsing it:

```typescript
export function renderCitationsAsHtml(
  input: string | ParsedCitationResult,
  options?: HtmlRenderOptions,
): HtmlOutput {
  const parsed = typeof input === "string" ? parseCitationResponse(input) : input;
  // ... split-and-rebuild
}
```

This keeps the API simple for callers who don't need the intermediate `ParsedCitationResult`.
